# Contributing to Apex Pulse

Thank you for your interest in contributing to the Apex Pulse project. We especially welcome contributions from the security research community.

## How to Contribute

### Security Research & Responsible Disclosure

Security is the core of what we build. If you discover a vulnerability:

1. **Do not** open a public issue
2. Email **security@apexelectronics.com** with details
3. Include: description, reproduction steps, potential impact, and your contact info
4. We will acknowledge within 48 hours and provide a remediation timeline
5. Credit will be given in our public security advisory (with your permission)

### Documentation Improvements

1. Fork this repository
2. Create a branch: `git checkout -b docs/your-improvement`
3. Make your changes in the `docs/` directory
4. Open a pull request with a clear description

### General Guidelines

- Keep PRs focused — one change per PR
- Follow existing document formatting
- All contributions are subject to the Apache 2.0 license

## Code of Conduct

We expect all contributors to be respectful and professional. Harassment of any kind will not be tolerated.
